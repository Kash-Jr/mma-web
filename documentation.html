MMA Web Project - Technical Documentation

Living technical documentation for the Multimedia Applications (MMA710S) portfolio website.

1. Target Audience

This documentation is explicitly written for:

  • Evaluators/Lecturers: Needing to understand how technical requirements (e.g., animations, audio, styling) were implemented without restricted technologies (like frames or deprecated Flash).
  • Future Maintainers/Students: Reviewing the source code for reference on vanilla JavaScript component implementation (Custom Audio Player, Canvas Watermarking, Image Slider).
Note: This document assumes a working knowledge of HTML5, CSS3, and Vanilla DOM manipulation.

2. Glossary

TermDefinition
Canvas WatermarkerThe JavaScript module in index.html that draws profile images onto an HTML5 <canvas> and overlays the student ID text.
Custom Audio PlayerThe UI wrapper in education.html that controls the hidden native <audio> element via JavaScript.
Vertical Slider AnimationThe 2-column image animation in courses.html utilizing CSS transitions and JavaScript setInterval (sliding up/down).
Content LayerThe primary CSS class (.content-layer) used universally across all pages to host page-specific information above the background.

3. System Boundaries & Setup

The application is a purely static, client-side website. It has no build step, no external APIs, and no backend dependencies.

Dependencies & Environment

  • Runtime: Any modern browser (Chrome 90+, Edge 90+, Firefox 88+, Safari 14+).
  • Resolution: Optimized for 1024x768 desktop view, responsive down to 320px.
  • External Libraries: None. 100% Vanilla JS and CSS.

Directory Structure Constraints

For the site to function locally, the following directory structure must be strictly maintained relative to the HTML files:

/images/   # Contains .jpeg and .jpg assets
/audio/    # Contains "Education Background Audio.ogg"
/MyLook-1.css # Global stylesheet

4. Critical Components & Configurations

4.1 Canvas Watermarker (index.html)

Instead of hardcoding watermarks into images, index.html dynamically draws the image and text onto a canvas. This requires matching the data-member-name attribute to the memberImageMap.

// Exact Configuration from index.html (Lines 237-244)
const memberImageMap = {
    'Senia Shipepe': 'images/Senia seated.jpeg',
    'Phrancina Ndjambula': 'images/Phrancina seated.jpeg', 
    'Faith Hamutenya': 'images/Faith seated.jpeg', 
    'Andris Kaishungu': 'images/Andris seated.jpeg',
    'Alvine Murangi': 'images/Alvine seated.jpeg'
};

4.2 Custom Audio Player (education.html)

The native audio element is hidden (display: none implicitly via custom controls). The custom UI binds to the native element events (loadedmetadata, timeupdate, ended).

Critical Element: The source file must remain an OGG format as defined in the source:

<audio id="edu-audio" preload="metadata">
    <source src="audio/Education Background Audio.ogg" type="audio/ogg">
</audio>

4.3 Vertical Slider Animation (courses.html)

The animation relies on 15 image paths hardcoded into the imageSources array in courses.html. Missing images will result in blank slides.

// Timing configuration
// Left column slides UP every 3000ms
setInterval(..., 3000);

// Right column slides DOWN every 3000ms, offset by 1500ms
setTimeout(function () { setInterval(..., 3000); }, 1500);

5. Architecture Decision Records (ADRs)

ADR 1: Dynamic Canvas Watermarking vs. Pre-rendered Images

Context: The assignment requires all images to have a Student ID watermark (Arial, size 8).

Decision: Use HTML5 <canvas> on the home page (index.html) to draw the image and overlay the text dynamically via JS.

Tradeoffs: Increases client-side execution slightly. Requires a fallback (implemented) if the image path fails. Prevents the need to destructively edit the original image assets, maintaining clean source files.

ADR 2: Vanilla JS Animation vs. Adobe Flash

Context: The brief requested a "Flash animation", but Adobe Flash is globally deprecated and blocked by all modern browsers.

Decision: Implemented a continuous, two-column vertical sliding animation using pure CSS (transforms) and Vanilla JavaScript in courses.html.

Tradeoffs: Ensures 100% compatibility across all modern devices (including mobile/iOS where Flash never worked) while strictly adhering to the spirit of the "animation" requirement without using video formats.

ADR 3: Custom Audio UI vs Native controls

Context: Native audio controls look vastly different across Chrome, Firefox, and Safari, potentially breaking the site's design language.

Decision: Built a custom audio player using SVG icons and JS event listeners mapped to the hidden HTMLAudioElement.

Tradeoffs: Adds ~130 lines of JS/CSS to education.html, but guarantees visual consistency with the #1a5f7a and #ff6b6b color palette.

6. Failure Modes & Troubleshooting

Do not assume the "happy path". Review these common failure states if the application breaks:

Symptom Root Cause Recovery / Resolution
index.html: Grey boxes appear instead of profile images in the Collage. Image paths in memberImageMap are incorrect, or files are missing. Canvas onerror fallback triggered. Verify that images/Senia seated.jpeg (and others) exactly match the case and spacing of the actual files in the images/ directory.
index.html: Canvas images fail to load when viewed via file:// protocol. Strict CORS policies in some browsers block canvas image manipulation from local file systems. Serve the directory over a local web server (e.g., VS Code Live Server, or npx serve), or test in Firefox which is more lenient with local file canvas drawing.
education.html: Audio player time says 0:00 / 0:00 and won't play. The browser cannot decode the .ogg file, or the file is missing from the audio/ directory. Ensure the file is named exactly Education Background Audio.ogg. If testing on older Safari versions, OGG may not be supported (consider providing an MP3 fallback source if required by the evaluator).
courses.html: Animation columns have blank gaps while sliding. One or more image paths in the imageSources array are invalid (404 Not Found). Check the browser console (F12) for 404 errors. Ensure all 15 images listed in the array physically exist in the images/ folder.
All Pages: Layout is completely broken, background is white. The CSS file is not loading. Ensure MyLook-1.css is in the exact same directory as the HTML files. Check case sensitivity (Linux/Unix servers are case-sensitive).
